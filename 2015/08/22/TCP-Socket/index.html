TCP Scoket

  • Network
  • Socket
  • TCP

今天我们讨论的是关于TCP/IP网络中两个进程之间相互作用的主机模式，即Client/Server Model。
CLient和Server端典型场景如下：
1、Server Startup
2、Client Startup,then connects to the Server
3、Client send a request to Server
4、Server process the request,and sends a reply back to the Client
5、Loop 3 and 4 steps…(data transmit)
6、Client close its end of the connection,send a end-of-file notification to Server
7、Server close its end of the connection (Terminate or Wait for a new Client connection)

那么上述场景下，socket是怎么工作的呢？

总体交互图示：

1、socket()函数


作用：用于创建一个socket descriptor，唯一标识一个socket（这是active socket）。关于socket具体内容可以参考XXXXXX
描述：socket执行成功后，会返回一个socket descriptor，里面包含了protocal family(such as IPv4,IPv6)和socket type(such as stream,datagram)。socket descriptor存在于address family（AF_XXX）中，没有一个具体的地址，即没有protocol address，所以必须要调用bind()函数赋值地址，或者当调用connect()或者listen()函数时，kernel自动随机分配一个ephemeral port。

2、bind()函数


作用：给socket分配一个local protocol address
描述：bind()分配的protocol address是指定的protocol决定的，可能会分配32位的IPv4或者128位的IPv6，同时会分配16位的TCP端口号或者是UDP端口号。
一般情况下，client端不用调用bind()函数，这样kernel会随机分配一个port和系统不冲突，以及分配source IP address，所以，你每次调用connect()函数后，你会发现绑定的端口不同。其实你也可以再client端调用bind()进行连接，然而这样的话，每次调用connect()函数连接server时，绑定的端口都是自己指定的那个端口。
然而，server是必须要使用bing()函数进行绑定IP和Port的，因为client连接server总需要知道目的地吧，即server的IP和Port，不然不知道目的地，叫client怎么连接。

3、connect()函数


作用：建立与TCP Server的连接
描述：client不用调用bind()函数，直接调用connect()函数，kernel会自动分配一个port和source IP。

4、listen()函数


作用：将socket从CLOSED state转换到LISTEN state，并监听client的socket
描述：传入的第一个socket参数是个passive socket，第二个参数是socket的最大排队数。
要理解第二个参数的最大排队数，要理解内核的排队情况，内核中包括incomplete queue和completed queue两个队列。incomplete queue里排队的是三次握手中的SYN_RCVD STATE，completed queue里排队的则是ESTABLISHED STATE。
具体根据图示来分析一下两个队列：
当开始三次握手时，Client端发送SYN，Server端接收到后，就create an entry on incomplete queue，然后返回SYN的ACK确认，以及Server端的SYN。直到再次收到Client发回来的ACK确认，之前在incomplete queue中创建的entry将被移动到completed queue的末端。当进行调用accept()方法后，将会将completed queue的entry加入进程。否则，假如completed queue是空的，那么accept()这个进程将一直处于sleep状态，直到有entry。
listen函数中的第二个参数backlog就是incomplete queue和completed queue两个队列的最大值设置。

5、accept()函数


作用：接收client发过来的请求。
描述：accept()被调用的时机是：当completed queue中有有排队的connection时。accept被调用将会传入事先调用socket()函数产生的listen socket，这个listen socket在server整个生命周期内都是存在的。accept()被调用后将返回一个connected socket，这是kernel自动创建的一个socket descriptor，这个socket descriptor和listen socket不同的是，当它完成对client端的连接服务之后，它将close掉，而listen socket还存在。

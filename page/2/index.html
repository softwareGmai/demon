K-Means算法

最近看论文，经常接触一些聚类算法，其中就有提到k-means算法，所以就自己归纳整理了一下。
首先搞清楚“聚类”（Clustering）和“分类”（Classfication）是不一样的。
分类是事先知道了有N类，你要把样本数据归类到这N类里面。比方说，有两个类别，苹果和梨，我们有一堆苹果和梨乱混合在那里，需要我们归类。这个就是分类问题。
而聚类呢，是你根本不知道样本数据是能分成什么类别的，只有通过自己学习自动归类，统一类别的样本数据就更相似。

聚类的第一步呢，往往是将原始数据映射成N维的空间向量，然后做处理的。
怎么映射呢？
假设有m个样本集合，每个样本为(X1,X2,…,Xn)，即每个样本有n个特征。这样的话，每个Xi都是m维的向量。

接下来，正式介绍下K-means算法。
K-means算法是一种聚类方法，主要的目的是将再给定分类组数k（k < n）的情况下，将原始数据分成k类，表示为S={S1,S2,…,Sk}，即对以下表达式求最小值：

其中ui是Si的平均值。

具体算法步骤如下：
1、随机在图中取k个种子点（k是给定的分类组数）
2、计算图中所有点到k个种子点的距离。如果点A离种子点Si的距离最近，则点A属于点群Si。
3、根据聚类的结果，重新计算每个点群的中心
4、将图中所有点根据新的中心重新聚类
5、重复3、4两个步骤，知道中心点不再变化

TCP Scoket

今天我们讨论的是关于TCP/IP网络中两个进程之间相互作用的主机模式，即Client/Server Model。
CLient和Server端典型场景如下：
1、Server Startup
2、Client Startup,then connects to the Server
3、Client send a request to Server
4、Server process the request,and sends a reply back to the Client
5、Loop 3 and 4 steps…(data transmit)
6、Client close its end of the connection,send a end-of-file notification to Server
7、Server close its end of the connection (Terminate or Wait for a new Client connection)
